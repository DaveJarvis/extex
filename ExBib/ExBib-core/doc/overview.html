Objectives

The development of bcd has been lead by several objectives.
  1. The result of bcd in compatibility mode should imitate as much as possible the behaviour of BibTeX -- for runs without errors or warnings. This should be valid for the output file only. bcd should not be concerned about the exact reproduction of tracing messages or the contents of the log file.
  2. The compatibility mode should not be the default. The default mode should be the bcd mode, where certain extensions are provided and some curious design decisions of BibTeX are not imitated.
  3. One objective of bcd was to provide a clean design which would make it easy to understand and enhance the program. As a result the whole program has been structured into several components which are tied togeter with interfaces and factories.
  4. bcd should be configurable. This means that it should be possible to use different implementations of some of the core components. This could be used for example to mimic the behaviour of BibTeX 0.99c or BibTeX 1.0 (as described in a TUGboat article).
  5. As implementation language Perl has been used for a first prototype. Then the implementation language has been switched to Java to make use of the Unicode features of this languages as well as to be closer to a potential successor of TeX.
  6. As much as possible the inherent knowledge about TeX shoud be eliminated.
  7. An easy update path sould be provided for BibTeX users: Old documents should be processable with the same results. Migration scripts should provided whenever neccesary.

Extensions

The following extensions can be envisaged:
  1. Implement the new features of BibTeX 1.0.
  2. Add new primitives to the BST language.
  3. Extend the syntax for BibTeX files
    • Allow structured description of names.
    • Make provisions for multi-language entries
    • Allow structured documents: i.e. articles in jounals -- without crossref

Bugfixes

The following defects of BibTeX should be cured:
  1. No restriction on the sequence of entries in the database. I.e. referenced entries may preceed or follow the referencing entries.
  2. Better sorting has to be provided taking into accout the various national conventions for sorting.
  3. No limit to some magical internal constants -- like the line length.
  4. Bugfix: flush the output buffer at the end of the program and not only when newline$ is encountered.
