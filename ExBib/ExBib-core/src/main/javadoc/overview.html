εχBib is the BIBTeX Compatible Database. It is an attempt to provide an implementation of a bibliographic database to be used with LATeX and friends.

Objectives

The development of εχBib has been lead by several objectives.
  1. εχBib is the attempt to provide a Bibliographic Database which can be used as a plug-in replacement for BIBTeX in its command line version.
  2. The result of εχBib in compatibility mode should imitate as much as possible the behavior of BIBTeX – for runs without errors or warnings. This should be valid for the output file only. εχBib should not be concerned about the exact reproduction of tracing messages or the contents of the log file.
  3. The compatibility mode should not be the default. The default mode should be the εχBib mode, where certain extensions are provided and some curious design decisions of BIBTeX are not imitated.
  4. εχBib should be configurable. This means that it should be possible to use different implementations of some of the core components. This could be used for example to mimic the behavior of BIBTeX 0.99c or BIBTeX 1.0 (as described in a TUGboat article).
  5. One objective of εχBib is to provide a clean design which makes it easy to understand and enhance the program. As a result the whole program has to be structured into several components which are tied together with interfaces and factories.
  6. As implementation language Perl has been used for a first prototype. Then the implementation language has been switched to Java to make use of the Unicode features of this languages as well as to be closer to a potential successor of TeX.
  7. The core components should provide a interface which make it possible to use them from within a typesetting engine without the need to use external aux files. Several passes within one run have to be taken into account.
  8. As much as possible the inherent knowledge about TeX should be eliminated.
  9. An easy update path should be provided for BIBTeX users: Old documents should be processable with the same results. Migration scripts should provided whenever necessary.

Extensions

The following extensions can be envisaged:
  1. Implement the new features of BIBTeX 1.0.
  2. Add new primitives to the BST language.
  3. Extend the syntax for BIBTeX files
    • Allow structured description of names.
    • Make provisions for multi-language entries
    • Allow structured documents: i.e. articles in journals – without crossref
  4. Integrate parsers for other bibliography formats
    • DocBook
    • refer
    • Endnote

Bugfixes

The following defects of BIBTeX should be cured:
  1. No restriction on the sequence of entries in the database. I.e. referenced entries may precede or follow the referencing entries.
  2. Better sorting has to be provided taking into account the various national conventions for sorting.
  3. No limit to some magical internal constants – like the line length.
  4. The line breaking should not insert percents (%) but break at whitespace only.
  5. Bugfix: flush the output buffer at the end of the program and not only when newline$ is encountered.
