The Directory Structure

This module follows the Maven directory structure. The inputs for compilation are somewhere under the directory src. The compiled results can be found in the directory target. 

 -+— src/
  |    +— main/
  |         +— image/
  |         |    +— ...
  |         +— tex/
  |              +— manual/
  |                   +— ...
  +— target/
  +— pom.xml

Some common files are taken from a similar directory structure in the parent directory: 

 -+— src/
       +— main/
            +— bibtex/
            |    +— references.bib
            +— texinputs/
                 +— bibtex/
                 |    +— alpha.bst
                 +— makeindex/
                 |    +— doc.ist
                 +— extex.doc.cls

Creating the Developer's Guide

The εχTEX developer's guide can be created in this directory with the help of Maven2. As a prerequisite the Maven module ExBib-Main needs to be accessible at least from the local repository. This is accomplished by 

  # cd ../../ExBib
  # mvn install

This command requires that Maven2 is properly installed on your system and the executable can be found on the search path. In addition a internet connection is required to download all artifacts not present locally.

Then the pdf can be generated with the following command: 

  # mvn compile

After a successful completion of the command the document can be found as the file target/ExTeX-developers.pdf.

If the build fails the verbose log messages of Maven can be used to determine the problem. Usually just reading the messages leads you into the right direction.
